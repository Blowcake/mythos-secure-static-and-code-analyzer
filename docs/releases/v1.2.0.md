# Release Notes - au3Mythos v1.2.0

We are pleased to announce the release of **au3Mythos v1.2.0**. This is a major update focused on false-positive hardening, file encoding stability, expanded test coverage, and compiler toolchain optimizations.

## Highlights & Features

- **False-Positive Hardening**: Substantial improvements implemented based on detailed reports and counter-fixtures provided by AspirinJunkie. Specific improvements include resolving scope checks in terminal branch control flows (such as `Local $iErr` in `JSON.au3` conditionals).
- **Encoding-Safe Source Loading**: Lossless reading of AutoIt files with full support for UTF-8 (with or without BOM), UTF-16 (Little Endian / Big Endian), and ANSI (Windows-1252) formats. Any malformed source files now cleanly trigger a `Source Read Error` and return a non-zero exit code instead of silently passing.
- **Project-Vendored UDF Dependency**: The verified stable UDF version of `JSON.au3` (v0.5.1 by AspirinJunkie and Sven Seyfert) has been vendored directly into the project repository at `third_party/autoit-json-udf/JSON.au3` under WTFPL license terms.
- **Measured Performance Improvements**: A reduction in overall analyzer runtime of **8.93%** achieved via optimized caching of lexer operations and fast keyword guards on frequent regex scans.

## Changelog Details

### Fixed
- Supported conditional top-level declarations like `If Not IsDeclared(...) Then Global` and single-line localized statements such as `Then Local`.
- Properly recognized variables used inside `ElseIf`, `Case`, `Case ... To`, and `Until` blocks as read references.
- Tailored `StringRegExp` checks to inspect flag `3` for array structures.
- Distinguished redeclared variables via `Dim` from brand new declarations.
- Exempted writable `ByRef` parameters from being flagged as dead stores.
- Resolved false duplicate declaration alerts in terminal control flow branches.
- Harmonized duplicate case value diagnostics across top-level and function scopes.

### Performance & Benchmarks
- Total runtime across 20 isolated test processes reduced from **158,541.568 ms** to **144,378.893 ms** (-14,162.675 ms, or **8.93%**).
- 20 out of 20 benchmarked preprocessed outputs deliver identical results.
